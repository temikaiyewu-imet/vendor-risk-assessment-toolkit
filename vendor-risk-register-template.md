## How to use this register

- Add one row per vendor after completing the assessment checklist
- Update the risk rating and status at each reassessment
- Flag any vendor whose residual risk has increased since last review
- Escalate Critical-rated vendors to senior risk ownership immediately

## Field definitions

| Field | Definition |
|---|---|
| Tier | 1 = Critical, 2 = Significant, 3 = Low Risk (see vendor-tiering-guide.md) |
| Data Types | e.g. Personal Data, Financial Data, Health Data, No Personal Data |
| Inherent Risk | Low / Medium / High / Critical — before controls (see scoring matrix) |
| Control Effectiveness | Weak / Moderate / Strong / Very Strong |
| Residual Risk | Low / Medium / High / Critical — after controls |
| Status | Active / Under Review / Suspended / Offboarding |
| Open Issues | Any remediation items or conditions outstanding from last assessment |
